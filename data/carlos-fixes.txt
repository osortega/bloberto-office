BUG 1 — Thought bubble double requestAnimationFrame (STRIKE 8)
--------------------------------------------------------------
Location: Character component, ~line 1473-1474 (useEffect for thought bubble)

Problem: Two nested requestAnimationFrame() calls were made without storing the
returned handles, so the animations could not be cancelled on cleanup. This risked
setState calls on unmounted components.

Fix:
- Added two new useRef() declarations at the top of Character:
    const thoughtRaf1Ref = useRef(null)
    const thoughtRaf2Ref = useRef(null)
- Stored both rAF handles:
    thoughtRaf1Ref.current = requestAnimationFrame(() => {
      thoughtRaf2Ref.current = requestAnimationFrame(() => ...)
    })
- Added cancelAnimationFrame(thoughtRaf1Ref.current) and
  cancelAnimationFrame(thoughtRaf2Ref.current) in the cleanup return of
  the same useEffect.

BUG 2 — Post-huddle timestamp freezes (STRIKE 3)
-------------------------------------------------
Location: ConferenceTable component, ~line 1027

Problem: The setTick interval (used to keep "X minutes ago" labels live) only
ran while meetingWorkers.length >= 2. When a meeting ended the interval was
cleared and never restarted, so the "Last: names · X minutes ago" label froze.

Fix: Added a second useEffect with the inverse guard that keeps the tick
interval running when there is no active meeting (meetingWorkers.length < 2):
    useEffect(() => {
      if (meetingWorkers.length >= 2) return
      const id = setInterval(() => setTick(t => t + 1), 30_000)
      return () => clearInterval(id)
    }, [meetingWorkers.length])
